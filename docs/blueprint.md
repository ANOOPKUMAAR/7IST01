# **App Name**: NetAttend

## Core Features:

- Wi-Fi Zone Definition: Allow users to define specific Wi-Fi networks (SSID) as attendance zones. Store Wi-Fi network names.
- Automated Check-in/Check-out: Automatically mark check-ins when the device connects to a defined Wi-Fi network. Automatically mark check-outs upon disconnection.
- Attendance Log: Display a detailed log of attendance records, including check-in and check-out times, using tabular or calendar views.
- AI-Powered Anomaly Detection: Employ an AI tool to identify and flag unusual attendance patterns, like unexpected late check-ins or early check-outs, based on learned user behavior. Can flag for manual review.
- Admin Mode Switch: Allows a privileged user to modify all the data. User can only unlock if he knows the correct security code, set upon installation of the application
- Class-wise Attendance: Show attendance seperately for each subject

## Style Guidelines:

- Primary color: Clean blue (#42A5F5) to evoke a sense of reliability and professionalism.
- Background color: Light desaturated blue (#E3F2FD) for a clean and unobtrusive backdrop.
- Accent color: A slightly brighter and more saturated blue (#2979FF) to highlight key interactive elements and call-to-actions, such as buttons.
- Body and headline font: 'PT Sans' sans-serif for a modern and readable experience.
- Use simple, outlined icons for navigation and representing attendance status (e.g., check-in, check-out, absent).
- Employ a clean, card-based layout for displaying attendance records. Keep the interface minimal and intuitive.
- Use subtle transitions and animations to provide feedback on user interactions, like check-in/check-out confirmations.
- Attendance below 75%: Red to indicate critical status.
- Attendance between 75% and 85%: Orange to indicate warning status.
- Attendance above 85%: Green to indicate good status.
